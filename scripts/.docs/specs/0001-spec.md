## 1. Overview

`.local-dev-toolbox` is a cross-platform developer automation and environment setup toolkit targeting individual developers and small teams sharing a common workflow. It covers four areas:

- **Bulk Git repository management** ‚Äî fetch/pull many repos in parallel (Bash + PowerShell)
- **Dev environment bootstrapping** ‚Äî idempotent, category-driven tool installation and dotfile management (Bash)
- **Shell configuration** ‚Äî team-reusable `.bashrc`/`.zshrc` with graceful degradation for optional tools
- **Git hygiene** ‚Äî pre-commit secret scanning and commit-message ticket prefixing

The toolkit is designed to be consumed from a shared mount point (e.g. a Windows host repo accessible at `/mnt/d/Repos/.local-dev-toolbox` inside WSL) and to run safely multiple times on the same machine without side effects.

---

## 2. Goals and Non-Goals

### Goals

- **Idempotent by default.** Every script can be re-run safely. Install steps are guarded; dotfile links are not duplicated.
- **Graceful degradation.** Optional tools (fzf, eza, bat, zoxide) are detected at runtime. Their absence does not break the shell or fail scripts.
- **Self-contained.** No external library dependencies beyond the tools being installed. Scripts bootstrap themselves.
- **Cross-platform within scope.** Linux (Debian/Ubuntu), macOS, and WSL2 for Bash. Windows (PowerShell 5.1+) for PowerShell and batch scripts.
- **Team-shareable.** Shell configs contain no personal secrets. Secrets go in `.bashrc.local` (gitignored).
- **Documented per component.** Every script has a matching `.md` in `scripts/.docs/`.

### Non-Goals

- Not a dotfile manager (no Stow, chezmoi, or similar). Dotfiles are symlinked by `setup-distro.sh`.
- Not a package version manager (no pinned lock files for dev tools).
- Not a CI/CD pipeline ‚Äî scripts are for local developer machines only.
- Not a secrets manager ‚Äî `.bashrc.local` is the boundary; secrets never enter the repo.

## 4. Components

### 4.1 Bash Scripts

#### `_update-repos.sh`

Bulk Git fetch/pull for Linux/macOS. Scans a root directory for child folders whose names match a configurable prefix, then processes each repo in parallel using background subshells with poll-based job throttling.

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--root-path PATH` | `$HOME/repos` | Root directory to scan. **Effectively required** ‚Äî the default `Hydra` prefix matches nothing in a generic setup. |
| `--prefix PREFIX` | `Hydra` | Folder name prefix filter. **Effectively required** ‚Äî must be set to match the user's repo naming convention. |
| `--parallel N` | `4` | Maximum concurrent workers. |
| `--skip-dirty` | off | Skip repos with uncommitted changes instead of pulling. |
| `--stash-dirty` | off | Auto-stash before pull, pop after. Conflicts with `--skip-dirty`. |
| `--use-rebase` | off | Pull with `--rebase` instead of merge. |
| `--fetch-all-remotes` | off | Fetch all configured remotes, not just `origin`. |
| `--verbose` | off | Print per-repo detail including diffstat and timing. |

**Both `--root-path` and `--prefix` should be treated as required in practice.** The defaults exist only to satisfy the parser; they produce no-op runs on most systems.

**Parallel processing model**

```
main loop
  ‚îî‚îÄ for each repo: launch subshell in background ‚Üí write result to tmp file
       ‚îî‚îÄ poll loop (0.1s sleep) caps active workers at --parallel
main loop collects tmp files after all workers finish ‚Üí prints summary
```

Timing uses millisecond-precision `date +%s%3N`; falls back to second precision on older macOS (`date +%s`).

**Git output parsing**

A single `git status --porcelain -b` call per repo provides:
- Branch name and upstream tracking info (first line)
- Dirty state (any subsequent lines)

Error detection uses regex matching against git stderr: `error:|fatal:|CONFLICT|merge conflict|divergent branches`.

**Dirty-tree handling flow**

```
dirty repo detected
  ‚îú‚îÄ --skip-dirty  ‚Üí skip, report as SKIPPED
  ‚îú‚îÄ --stash-dirty ‚Üí git stash ‚Üí pull ‚Üí git stash pop ‚Üí report STASH_APPLIED or STASH_CONFLICT
  ‚îî‚îÄ (neither)     ‚Üí pull anyway, mark result as DIRTY
```

**Temp file cleanup:** `trap 'rm -rf "$TMPDIR_WORK"' EXIT` ensures no orphaned files on error or interrupt.

**Known limitation:** New remotes with no local tracking branch produce a git output pattern not matched by current error regexes. These repos will report as "no upstream" rather than as an error.

#### `setup-distro.sh`

Idempotent dev environment bootstrap for Debian/Ubuntu/WSL and macOS. Symlinks dotfiles, installs dev tools, and configures the shell.

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--only=CATEGORIES` | all | Comma-separated list of categories to run. |
| `--upgrade` | off | Re-install/upgrade tools even if already present. |
| `--dry-run` | ‚Äî | **Planned ‚Äî not yet implemented.** See ¬ß7. |

**Categories and dependency order**

The nine categories must be run in dependency order when using `--only`:

```
Tier 1 (no deps):   dotfiles  core  shell
Tier 2 (need core): cli  languages  web  containers  powershell
Tier 3 (need lang): cloud   (install_wrangler requires npm from languages)
```

`dotfiles`, `core`, and `shell` have no inter-dependencies and can run in any order within Tier 1. All Tier 2 categories depend on `core` providing the package manager. `cloud` depends on `languages` providing npm (for `wrangler`). When `--only` skips `languages`, `install_wrangler()` emits a warning and exits gracefully without failing the run.

**Category summary**

| Category | What it installs |
|----------|-----------------|
| `dotfiles` | Symlinks `~/.bashrc`, `~/.bash_profile` to repo; copies `.bashrc.local` from repo (or `.example` if absent); sets `DEV_TOOLBOX` in PATH |
| `core` | curl, wget, git, build-essential, unzip, jq, tree, htop, ncdu, tmux |
| `cli` | eza, bat/batcat, fzf, ripgrep, fd-find, zoxide, tldr, fastfetch, gitleaks |
| `shell` | starship, Nerd Fonts (Fira Code), zsh |
| `languages` | nvm ‚Üí Node.js LTS, python3 + pip + venv |
| `cloud` | Azure CLI, AWS CLI, Tailscale, wrangler (requires npm) |
| `web` | nginx, certbot |
| `containers` | Docker Engine (via official install script) |
| `powershell` | PowerShell 7 via package manager (snap on non-amd64) |

**Idempotency guard**

```bash
needs_install() {
  local cmd=$1
  cmd_exists "$cmd" && [[ "$UPGRADE" == "false" ]] && return 1  # skip
  return 0  # install
}
```

`Nerd Fonts` and some GitHub-release binaries use directory/package existence checks rather than `cmd_exists` because they install no binary to PATH.

**Platform detection**

```bash
_detect_platform() ‚Üí "macos" | "debian" | "unknown"
```

Package names are normalised by `pkg_name()`, which returns an empty string for packages that don't exist on macOS (build-essential, python3-pip, python3-venv, python3-certbot-nginx), causing them to be silently skipped. The macOS equivalents are provided by the OS or Homebrew under different names.

**Dotfile flow**

```
dotfiles category
  ‚îú‚îÄ ln -sf <repo>/.bashrc      ‚Üí ~/.bashrc       (symlink ‚Äî repo changes live immediately)
  ‚îú‚îÄ ln -sf <repo>/.bash_profile ‚Üí ~/.bash_profile (symlink)
  ‚îî‚îÄ .bashrc.local:
       ‚îú‚îÄ repo has .bashrc.local  ‚Üí cp to ~/.bashrc.local  (copy, not symlink ‚Äî contains secrets)
       ‚îî‚îÄ repo has no .bashrc.local ‚Üí cp .bashrc.local.example to ~/.bashrc.local
```

`~/.bashrc.local` is always a copy, never a symlink, because it may contain secrets.

#### `fc-rsync.sh`

General-purpose rsync wrapper for backing up a repo directory to another location. Respects `.gitignore`. Excludes `node_modules`, `.log`, `.tmp`, `.cache`.

**Parameters** (arg ‚Üí env var ‚Üí default)

| Arg | Env var | Default | Description |
|-----|---------|---------|-------------|
| `$1` | `RSYNC_SOURCE` | `/mnt/d/Repos/Hydra.OPT.Service` | Source path |
| `$2` | `RSYNC_DEST` | `/mnt/c/Backups/Hydra.OPT.Service` | Destination path |

The hardcoded defaults are illustrative only. In practice supply `$1`/`$2` or set the env vars.

**Known gaps:** No validation that source or destination paths exist before rsync runs. No dry-run flag. Failures from rsync propagate via exit code but produce no additional diagnostic message.

### 4.2 PowerShell Scripts

All PowerShell scripts target **PowerShell 5.1 minimum** and are tested on PowerShell 7+. Scripts use `Set-StrictMode -Version Latest` and `[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`. Unicode symbols use `[char]0xNNNN` escapes for cross-terminal consistency.

#### `_update-repos.ps1`

PowerShell equivalent of `_update-repos.sh`. Uses `RunspacePool` instead of `ForEach-Object -Parallel` for PowerShell 5.1 compatibility.

**Why RunspacePool over `ForEach-Object -Parallel`:** `-Parallel` was introduced in PowerShell 7. RunspacePool works on 5.1 and 7+, and provides explicit control over the thread pool size and script block isolation.

**Architecture**

```
RunspacePool (max $Parallel threads)
  ‚îî‚îÄ each runspace receives a self-contained script block + hashtable of args
       ‚îî‚îÄ returns a hashtable result (not PSCustomObject ‚Äî hashtables survive runspace boundaries cleanly)
main thread collects AsyncResult handles ‚Üí waits ‚Üí prints ordered summary
```

Pre-compiled `[regex]` patterns are constructed once in the caller and passed into runspaces to avoid repeated compilation overhead.

**Classes/constants:** `RepositoryStatus`, `PullStatus`, `RepositoryResultFactory` encapsulate status codes and result construction. This keeps runspace script blocks free of string literals.

**Flags** mirror the Bash version: `-RootPath`, `-Prefix`, `-Parallel`, `-SkipDirty`, `-StashDirty`, `-UseRebase`, `-FetchAllRemotes`, `-Verbose`.

#### `git-history.ps1`

Generates formatted release notes from `git log` output for a named time period.

**Periods:** `today`, `this_week`, `last_week`, `last_4_weeks`, `current_month`, `last_month`

**Output includes:**
- Ticket number extracted from commit message (regex: leading digits and dash patterns)
- Author initials
- Normalised message body (strips boilerplate, applies proper case)
- Optional ticket hyperlinks when a base URL is supplied

#### `fix-timestamps.ps1`

Batch-updates `CreationTime`, `LastWriteTime`, and `LastAccessTime` on files matching a given extension filter. Parameters via CLI args or environment variables.

#### `migrate-wsl-distro.ps1`

Migrates a WSL distribution to a new filesystem path via `wsl --export` / `wsl --import`. Supports `--skip-unregister` and `--skip-export` flags for partial re-runs. Color-coded progress output.

#### Windows Sandbox (`sandbox/`)

Full provisioning pipeline for a Windows Sandbox instance configured for Forecourt Service development.

**Entry point:** `setup-wsb.ps1` (launched by `_sandbox-config.wsb` on sandbox logon)

**Provisioning order:**
1. `setup-winget.ps1` ‚Äî installs tools via winget
2. `setup-chocolatey.ps1` ‚Äî installs from winget and a custom internal NuGet source (credentials via env vars, not in files)
3. `setup-nodejs.ps1` ‚Äî Node.js LTS
4. `setup-angular.ps1` ‚Äî Angular CLI at a pinned version

**Sandbox config (`_sandbox-config.wsb`):** Maps host `scripts/powershell/sandbox/` ‚Üí `C:\TEMP` in sandbox. Networking enabled. Clipboard enabled. vGPU disabled.

### 4.3 Batch Scripts

#### `_backup_fc-repo.bat`

General-purpose Windows xcopy backup script. Copies a repository directory to a backup location, **wiping the destination first** to produce a clean snapshot on every run. This is intentional ‚Äî incremental/diff behaviour is not the goal.

**Parameters** (arg ‚Üí env var ‚Üí default)

| Arg | Env var | Default | Description |
|-----|---------|---------|-------------|
| `%1` | ‚Äî | `Hydra.OPT.Service` | Repository folder name |
| `%2` | `BACKUP_SOURCE_ROOT` | `D:\Repos` | Root containing repos |
| `%3` | `BACKUP_DEST_ROOT` | `C:\Backups\Repos` | Root for backup output |
| `%4` | `BACKUP_EXCLUDEFILE` | `%~dp0.exclude.txt` | Path to xcopy exclusion list |

**Backup flow:**
```
rd /s /q "%DEST%"        ‚Üê full wipe of previous backup
mkdir "%DEST%"
xcopy /e /h /c /i /y /EXCLUDE:%EXCLUDEFILE%
```

Exits with xcopy's error level. No logging by default (commented log path provided as example).

---

#### `.exclude.txt`

General-purpose xcopy exclusion list for source code repositories. Excludes build artifacts, IDE metadata, test results, and package caches.

**Excluded paths/patterns:**

```
node_modules, build, obj, out
bin\x86, bin\x64\Debug\net462/net472/net481/net6.0/net8.0
bin\Debug\net462/net472/net481/net6.0/net8.0
.bin, .dist, .next, .nuxt, .angular
coverage, logs, log, TestResults, .packages
.vs, .idea, __pycache__
*.log, *.tmp, *.cache, *.user, *.suo, *.ds_store, thumbs.db
```

This file is general-purpose and not project-specific. Project-specific exclusions should be maintained separately.

### 4.4 Shell Configuration (Dotfiles)

#### `.bashrc`

Team-reusable interactive Bash configuration. Only activates for interactive shells (`[[ $- != *i* ]] && return`).

**Section layout (in order):**
`INIT` ‚Üí `SHELL OPTIONS` ‚Üí `HISTORY` ‚Üí `ENV` ‚Üí `FUNCTIONS` ‚Üí `ALIASES` ‚Üí `KEYBINDINGS` ‚Üí `PROMPT` ‚Üí `EXTERNAL` ‚Üí `LOCAL OVERRIDES`

**Key design decisions:**

- **Platform-aware clipboard:** Detects WSL (`clip.exe`), Wayland (`wl-copy`), X11 (`xclip`), macOS (`pbcopy`) at runtime.
- **Listing fallback chain:** `eza` ‚Üí `lsd` ‚Üí `ls`. The `ll`, `la`, `ls` aliases resolve to whichever is available.
- **History settings:** `HISTCONTROL=erasedups:ignoredups:ignorespace`. `ignorespace` suppresses commands prefixed with a space (useful for ad-hoc secrets in the shell; not widely documented).
- **XDG Base Directory:** `XDG_CONFIG_HOME`, `XDG_DATA_HOME`, `XDG_CACHE_HOME` set to standard paths.
- **`DEV_TOOLBOX`:** Set by `setup-distro.sh` dotfiles category; adds the repo's bash scripts dir to `PATH` for direct script invocation.

**Functions:**

| Function | Description |
|----------|-------------|
| `extract <file>` | Handles 10+ archive formats (tar.gz, zip, rar, 7z, bz2, xz, etc.) |
| `cd <dir>` | Overrides builtin ‚Äî auto-runs `ls` after changing directory |
| `myip` | Shows LAN IP (platform-aware route detection) and external IP via `curl` |
| `mkcd <dir>` | `mkdir -p` + `cd` |
| `bak <file>` | Copies file to `file.bak.YYYYMMDD` |
| `up [N]` | Navigates N directories up (`up 3` ‚Üí `cd ../../..`) |
| `pwdtail` | Prints last 2 path components |
| `gcom <msg>` | `git add -A && git commit -m` |
| `lazy <msg>` | `gcom` + `git push` |
| `gclean` | Deletes merged local branches |
| `search_files <pat>` | `rg` with fallback to `grep -r` |
| `cheat <cmd>` | Fetches cht.sh cheatsheet |
| `fe` | fzf file picker ‚Üí opens in `$EDITOR` (requires fzf + fd) |
| `fcd` | fzf directory picker ‚Üí `cd` (requires fzf + fd) |
| `fkill` | fzf process picker ‚Üí `kill` (requires fzf) |
| `fshow` | fzf git log browser (requires fzf) |

**Optional tool detection:** fzf functions are gated on `command -v fzf` checks. The `fe` and `fcd` functions additionally require `fd` ‚Äî both tools must be present. If fzf is absent, the functions are not defined.

**Aliases:** 70+ aliases covering navigation, listing (distro-aware `bat`/`batcat`), archives, monitoring, package management, git, Docker, networking. See [`scripts/.docs/bash/bashrc.md`](bash/bashrc.md) for the full list.

#### `.zshrc`

Functional equivalent of `.bashrc` for Zsh. Uses Zsh idioms throughout:
- `setopt` instead of `shopt`
- `autoload -Uz compinit && compinit` for tab completion
- `bindkey` / `zle -N` for key bindings
- `print` instead of `echo` in some functions

All functions, aliases, and tool-detection logic are equivalent to `.bashrc`.

#### `.bash_profile`

Minimal login shell bootstrap. Sources `/etc/profile` then `~/.bashrc`. No additional logic.

#### `.bashrc.local` / `.bashrc.local.example`

`.bashrc.local` is sourced at the end of `.bashrc` and `.zshrc` for personal overrides: secrets, machine-specific aliases, custom `PATH` entries. It is gitignored and never symlinked ‚Äî always a plain copy on the local machine.

`.bashrc.local.example` is the template committed to the repo. `setup-distro.sh` copies it to `~/.bashrc.local` when no existing `.bashrc.local` is found.

### 4.5 Git Hooks

Install by copying into `.git/hooks/` and marking executable:

```bash
cp scripts/git-hooks/pre-commit  .git/hooks/pre-commit
cp scripts/git-hooks/commit-msg  .git/hooks/commit-msg
chmod +x .git/hooks/pre-commit .git/hooks/commit-msg
```

#### `pre-commit`

Runs gitleaks against staged changes before every commit.

**Flow:**
```
SKIP_GITLEAKS=1 set?  ‚Üí exit 0 (bypass)
gitleaks not in PATH? ‚Üí warn, exit 0 (graceful degradation)
gitleaks exits 0?     ‚Üí exit 0 (clean)
gitleaks exits non-0? ‚Üí print remediation options, exit 1 (block commit)
```

**Remediation options presented to the user:**
1. Remove the secret from staged changes
2. Add `# gitleaks:allow` inline comment for known false positives
3. Add pattern to `.gitleaks.toml` `[allowlist]`
4. Set `SKIP_GITLEAKS=1` (sparingly)

**Config path:** The hook must pass `--config` pointing to `scripts/git-hooks/.gitleaks.toml` resolved relative to the repo root. Without `--config`, gitleaks looks for `.gitleaks.toml` at the working directory root and silently falls back to defaults if not found. See ¬ß7.

#### `commit-msg`

Auto-prefixes commit messages with the ticket number extracted from the current branch name.

**Supported branch patterns:**

| Branch pattern | Extracted ticket | Example output |
|---------------|-----------------|----------------|
| `feature/123-foo` | `123` | `123 - Original message` |
| `bugfix/456-bar` | `456` | `456 - Original message` |
| `hotfix/789-baz` | `789` | `789 - Original message` |

**Skip conditions (no prefix applied):**
- Branch is `main`, `master`, or `develop`
- Branch name does not match the ticket pattern
- Commit message already starts with digits
- Commit is a merge commit (message starts with `Merge `)
- Detached HEAD state

The hook modifies the commit message file in-place (`$1` is the path to the message file).

## 5. Architecture and Design Decisions

### Parallel Processing

**Bash (`_update-repos.sh`):** Background subshells with a poll loop (0.1s sleep) to throttle active workers to `--parallel N`. Results written to temp files in `$TMPDIR_WORK`; collected after all workers complete. Order of completion is non-deterministic; summary is printed after all workers finish.

**PowerShell (`_update-repos.ps1`):** `RunspacePool` with a fixed thread count. `ForEach-Object -Parallel` was rejected because it requires PowerShell 7; RunspacePool works on 5.1+. Runspaces receive self-contained script blocks with no dependency on caller scope. Results returned as hashtables (serializable across runspace boundaries; PSCustomObjects are not reliable).

### Git Status Parsing

Single `git status --porcelain -b` call per repo. The first line encodes branch and upstream info; any additional lines indicate dirty state. This avoids multiple git subprocess invocations per repo and keeps the parser deterministic.

### Repository Discovery

Scripts scan `--root-path` for immediate child directories whose names match `--prefix`. Subdirectories are not recursed. The prefix filter is case-sensitive. Symlinked directories are included if the OS follows symlinks in directory listing (default on Linux/macOS).

### Dotfile Management

Symlinks for `.bashrc` and `.bash_profile` mean changes in the toolbox repo are immediately reflected in all shells without re-running `setup-distro.sh`. `.bashrc.local` is always a copy because it may contain secrets that should not be managed by the toolbox.

### Error Handling

- Bash scripts: `set -euo pipefail`. Subshell workers catch errors locally so one failed repo does not kill the main process.
- PowerShell scripts: `Set-StrictMode -Version Latest`. Runspace errors are captured and reported per-repo.
- Batch scripts: `xcopy /c` continues on error; final error level check at end.

### Output Encoding

ANSI colour codes are manual escape sequences (`\e[...m` in Bash, `` `e[...m `` in PowerShell). Unicode symbols use `\uNNNN` escapes (Bash) and `[char]0xNNNN` (PowerShell) for cross-terminal consistency rather than literal UTF-8 characters in source.

## 7. Known Gaps and Planned Improvements

These are confirmed gaps identified during spec review. Each item is actionable.

---

### GAP-001 ‚Äî `pre-commit` hook does not pass `--config` to gitleaks

**Impact:** `scripts/git-hooks/.gitleaks.toml` is silently ignored. Gitleaks falls back to its default ruleset with no custom allowlists.

**Fix:** Pass the config path explicitly, resolved from the repo root:

```bash
REPO_ROOT=$(git rev-parse --show-toplevel)
CONFIG="$REPO_ROOT/scripts/git-hooks/.gitleaks.toml"
gitleaks git --pre-commit --staged --verbose --config "$CONFIG"
```

If the config file is absent, the hook should warn but not fail (consistent with the existing gitleaks-absent behaviour).

---

### GAP-002 ‚Äî README states 9 install categories as 8

**Impact:** Misleading documentation. "Bootstrap...installs all dev tools across 8 categories" in the Quick Start section. `dotfiles` is the ninth category.

**Fix:** Update Quick Start to say "9 categories (dotfiles, core, cli, shell, languages, cloud, web, containers, powershell)".

---

### GAP-003 ‚Äî `setup-distro.sh` has no `--dry-run` flag

**Impact:** Users cannot preview what would be installed/changed without committing to the run.

**Expected behaviour of `--dry-run`:**
- Print each install step that would execute (with `[DRY RUN]` prefix)
- Do not call package managers, curl, or any install functions
- Symlink and copy operations in `dotfiles` category are also skipped
- `--dry-run` with `--upgrade` shows what would be upgraded
- Exit code 0 on clean dry run

---

### GAP-004 ‚Äî `commit-msg` hook does not exclude merge commits

**Impact:** Merge commits (message starts with `Merge branch '...'`) are incorrectly prefixed with a ticket number.

**Fix:** Add an early exit when the commit message begins with `Merge `:

```bash
MSG=$(cat "$1")
[[ "$MSG" == Merge\ * ]] && exit 0
```

---

### GAP-005 ‚Äî No `starship-theme` switch command

**Impact:** Users must manually copy theme files over `starship.toml`. No discovery of available themes.

**Expected behaviour:** A shell function `starship-theme` (defined in `.bashrc` and `.zshrc`) that:
- With no args: lists available themes from `$DEV_TOOLBOX/.config/starship/`
- With a theme name: copies that theme to `~/.config/starship.toml` and prints confirmation
- Example: `starship-theme tokyo-night`

---

### GAP-006 ‚Äî `.exclude.txt` contained project-specific entries

**Impact:** File was not general-purpose as intended. Project-specific paths (`OPTService - Moved to LRM repo\`, `OPT.Common - Moved to LRM repo\`) were committed.

**Fix:** Remove those lines. The file should contain only framework/tooling patterns applicable to any project. **Status: to be applied.**

---

### GAP-007 ‚Äî `fc-rsync.sh` has no source/destination validation

**Impact:** If source or destination paths don't exist, rsync fails with a cryptic error.

**Planned improvement:** Add existence checks before invoking rsync:
```bash
[[ -d "$SOURCE" ]] || { echo "Error: source not found: $SOURCE"; exit 1; }
```

---

### GAP-008 ‚Äî `_update-repos.sh` and `_update-repos.ps1` treat `--prefix` as optional

**Impact:** Confusing UX ‚Äî the default prefix `Hydra` matches no repos on a generic machine. Script runs silently with zero repos processed.

**Planned improvement:** Print a warning when zero repos are found and suggest setting `--prefix`. A stricter option: require `--prefix` explicitly and fail if not supplied.

---

## 8. Conventions

### Adding a New Script

1. Place under `scripts/<area>/` using the existing naming pattern.
2. Add a matching documentation file at `scripts/.docs/<area>/<script-name>.md`.
3. Update `README.md` scripts table.
4. Bash scripts: begin with `set -euo pipefail` and a usage/help block.
5. PowerShell scripts: begin with `Set-StrictMode -Version Latest` and `[CmdletBinding()]`.
6. No external library dependencies ‚Äî scripts must be self-contained.

### Documentation Files

Each `.md` in `scripts/.docs/` should cover:
- **Purpose** ‚Äî one paragraph
- **Usage** ‚Äî command examples with all flags
- **Parameters** ‚Äî table of all flags/args with defaults
- **Behaviour** ‚Äî key flows, edge cases, error handling
- **Platform notes** ‚Äî where applicable

### Secret Handling

- Secrets never go in tracked files. `.bashrc.local` is gitignored.
- The pre-commit gitleaks hook is the safety net. Install it in every repo that uses these dotfiles.
- False positives: use `# gitleaks:allow` inline or add to `scripts/git-hooks/.gitleaks.toml`.

### Starship Theme Workflow

Until GAP-005 is resolved: copy the desired theme file over `~/.config/starship.toml`. Available themes are in `scripts/bash/.config/starship/`. The active committed theme (`starship.toml`) is Nord-based.

### Git Hook Installation

Hooks are not auto-installed by `setup-distro.sh`. They must be copied manually into each target repo's `.git/hooks/`. This is intentional ‚Äî hook installation is a per-repo decision.

### PowerShell Compatibility Baseline

All PowerShell scripts must run on **PowerShell 5.1**. Do not use `ForEach-Object -Parallel`, `?? =`, or other PS7-only syntax. Test against both 5.1 and 7.
